Send one email through a configured account. Write the body once in `markdown` — it renders to inline-styled HTML plus a plain-text alternative, and a paragraph that is exactly one link becomes a button (the "click to sign in" line costs nothing to author) — or pass explicit `html`/`text`, which win per part. Attachments accept in-memory media (`{bytes,mime}`, what the AI generation ops output), blob references (`{blobId}`, what `store.blob.put` returns — mod-store resolves the bytes), or literal files (`{filename,content}`). With nothing wired to `account` it uses the `"default"` account; wire an `email.account` node (or a bare name string) to pick another. `from` defaults to the account's From and can be overridden per message. Credentials resolve at send time from the vault or env — they never ride the graph.
