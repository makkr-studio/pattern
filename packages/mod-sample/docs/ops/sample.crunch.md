Burns CPU on purpose (naive recursive fibonacci) to plant a visibly long, busy span on the Runs waterfall, a demo and load-test stand-in. Being `cpuHeavy`, it nudges the editor to suggest the workflow's Offload flag so it runs on the worker pool and keeps the host loop free; pair it with an idle `core.time.delay` (as `sample.replay` does) to contrast a busy span against an idle one.
