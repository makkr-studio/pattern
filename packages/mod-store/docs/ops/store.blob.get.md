Read a blob as a streaming `bytes` output plus `meta`, `found`, and ready-to-serve `headers`/`status`. Wire `bytes`/`headers`/`status` straight into a `boundary.http.response` (mode chunked) and you have a download route, which is the shipped `GET /store/blobs/:id`. On a miss it returns `found:false` with a 404 status and an empty stream, so the same wiring serves the not-found case. Keep `bytes` streaming so large files never fully buffer; reach for `store.blob.delete` to remove one.
